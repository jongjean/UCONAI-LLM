# 📝 포스팅툴 사용 가이드

## 🎯 개요

**포스팅툴**은 한국ESG학회 웹사이트의 메인페이지 히어로 슬라이더를 쉽고 빠르게 관리할 수 있는 관리자 전용 도구입니다.

모바일에서도 카카오톡처럼 간편하게 사용할 수 있으며, AI 편집 기능과 버전 히스토리 관리를 제공합니다.

---

## 🚀 시작하기

### 1. 접근 방법

#### 방법 A: 마이페이지에서
1. 웹사이트에 **로그인**
2. **마이페이지** 이동
3. **관리자 메뉴** 섹션의 **"포스팅툴"** 버튼 클릭

#### 방법 B: 직접 URL
- **포스팅툴**: `/pages/admin/posting-tool.html`
- **히스토리 관리**: `/pages/admin/history-manager.html`

---

## 📱 핵심 기능

### 1️⃣ 슬라이드 편집

포스팅툴에서 **3개의 슬라이드**를 동시에 관리할 수 있습니다.

#### 편집 가능한 요소
- ✅ **이미지**: 파일 업로드 또는 URL 입력
- ✅ **제목**: 슬라이드 메인 제목
- ✅ **설명**: 슬라이드 설명 텍스트
- ✅ **버튼 텍스트**: CTA 버튼 텍스트 (예: "자세히 보기")
- ✅ **버튼 링크**: 클릭 시 이동할 페이지 경로

#### 이미지 업로드 방법
1. **파일 업로드**
   - 이미지 영역 클릭 → 파일 선택
   - 권장 크기: `1920x1080px`
   - 최대 파일 크기: `5MB`
   - 지원 형식: JPG, PNG, WebP

2. **이미지 URL 입력**
   - "이미지 URL 입력" 필드에 직접 입력
   - 예: `https://example.com/image.jpg`

#### 슬라이드 액션
- 🪄 **AI 편집**: 텍스트 가독성 향상 및 ESG 키워드 최적화
- 👁️ **미리보기**: 실제 화면과 동일한 미리보기 제공
- 🔄 **초기화**: 슬라이드를 초기 상태로 되돌림

---

### 2️⃣ AI 편집 기능

AI 편집 버튼을 클릭하면 다음 작업이 자동으로 수행됩니다:

✨ **자동 개선 사항**
- 중복 공백 제거
- 텍스트 가독성 향상
- ESG 키워드 최적화
- 버튼 텍스트 표준화

> 📌 **주의**: 원래 의미는 절대 변경되지 않습니다.

---

### 3️⃣ 버전 저장 및 관리

#### 저장 방법
1. 3개 슬라이드 편집 완료
2. **"변경사항 저장"** 섹션으로 이동
3. **버전 제목** 입력 (필수)
   - 예: `🎉 월드ESG포럼 2025`
   - 예: `🏠 기본 메인페이지`
   - 예: `📰 최신 ESG 뉴스`
4. **상세 설명** 입력 (선택)
5. **"저장 및 게시"** 버튼 클릭

#### 저장 시 효과
- ✅ 메인페이지에 **즉시 반영**
- ✅ 히스토리에 **버전 저장**
- ✅ 이전 버전으로 **언제든지 복구 가능**

---

### 4️⃣ 히스토리 관리

히스토리 관리 페이지에서 저장된 모든 버전을 확인하고 관리할 수 있습니다.

#### 주요 기능
- 📋 **버전 목록**: 최신순으로 정렬된 모든 버전
- 🔍 **검색**: 제목, 설명, 작성자로 검색
- 👁️ **미리보기**: 선택한 버전의 전체 슬라이드 미리보기
- ✅ **버전 적용**: 이전 버전을 메인페이지에 적용
- ✏️ **수정하기**: 선택한 버전을 포스팅툴에서 편집
- 🗑️ **삭제하기**: 불필요한 버전 삭제

#### 버전 관리 흐름
```
1. 히스토리 목록에서 버전 클릭
   ↓
2. 오른쪽 미리보기 영역에 슬라이드 표시
   ↓
3. 원하는 액션 선택
   - "이 버전 적용하기" → 메인페이지 즉시 변경
   - "수정하기" → 포스팅툴로 이동
   - "삭제하기" → 버전 삭제
```

## 🔧 기술 구조

### 데이터 저장
- **LocalStorage** 기반 (프론트엔드)
- 키: `esg_hero_slides` (현재 슬라이드)
- 키: `esg_main_history` (버전 히스토리)
- 최대 50개 버전 저장

### 주요 파일
```
📁 pages/admin/
  ├── posting-tool.html      # 포스팅툴 페이지
  └── history-manager.html   # 히스토리 관리 페이지

📁 js/
  ├── posting-tool.js        # 포스팅툴 로직
  ├── history-manager.js     # 히스토리 관리 로직
  └── main.js                # 메인 슬라이더 동적 렌더링

📁 css/
  └── posting-tool.css       # 포스팅툴 스타일
```

## 🚨 주의사항

### ⚠️ 필수 확인 사항
1. **버전 제목 필수**: 저장 시 반드시 버전 제목 입력
2. **이미지 크기**: 5MB 이하 권장
3. **현재 버전 삭제 불가**: 다른 버전 적용 후 삭제 가능
4. **데이터 백업**: 중요한 버전은 Export 기능 사용 (향후 지원)

### 🔒 권한 관리
- **프론트엔드 시뮬레이션**: 현재는 로그인한 모든 사용자에게 권한 부여
- **실제 운영 시**: `user.role === 'SUPER_ADMIN'` 조건 추가 필요
- **서버 검증**: 백엔드 마이그레이션 시 서버 권한 검증 필수

## 🆘 문제 해결

### Q1: 저장 후 메인페이지에 반영이 안 돼요
**A**: 브라우저 캐시를 새로고침하세요 (Ctrl + F5 또는 Cmd + Shift + R)

### Q2: 이미지가 업로드되지 않아요
**A**: 
- 파일 크기가 5MB 이하인지 확인
- 이미지 형식이 JPG/PNG/WebP인지 확인
- 브라우저 콘솔에서 오류 메시지 확인

### Q3: 관리자 메뉴가 안 보여요
**A**:
- 로그인 상태 확인
- 마이페이지 새로고침
- 브라우저 콘솔에서 `localStorage.getItem('user')` 확인

### Q4: 히스토리가 삭제되었어요
**A**:
- LocalStorage 데이터 확인: F12 → Application → Local Storage
- 최대 50개 버전만 저장되므로 오래된 버전은 자동 삭제됨
- 중요한 버전은 수시로 백업 권장
