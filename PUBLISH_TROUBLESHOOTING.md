# 🚀 GenSpark Publish (게시) 문제 해결 가이드

## ⚠️ 현재 상황

**날짜**: 2025년 12월 30일 오후 10:45:42  
**문제**: 배포 후 빌드/배포 버튼이 비활성화됨  
**증상**: Publish 탭에서 버튼을 클릭할 수 없음

---

## 🎯 GenSpark Publish 시스템 이해

### **1. GenSpark의 배포 프로세스**

```
코드 작성 (샌드박스)
    ↓
[Publish 탭] → "Publish" 버튼 클릭
    ↓
빌드 시작 (Building...)
    ↓
배포 완료 (Deployed)
    ↓
실제 URL 생성
```

### **2. Publish 상태**

| 상태 | 설명 | 버튼 상태 |
|------|------|----------|
| **Not Published** | 아직 게시하지 않음 | ✅ Publish 버튼 활성화 |
| **Building** | 빌드 진행 중 | ❌ 버튼 비활성화 (로딩 중) |
| **Deployed** | 배포 완료 | ✅ Update 버튼 활성화 |
| **Failed** | 배포 실패 | ✅ Retry 버튼 활성화 |

---

## 🔍 버튼 비활성화 원인

### **원인 1: 빌드 진행 중** ⏳

**증상**:
- 버튼이 회색으로 비활성화
- "Building..." 또는 "Deploying..." 메시지 표시
- 진행 표시줄(Progress Bar) 표시

**해결**:
- ⏰ **기다리기**: 빌드가 완료될 때까지 대기 (보통 1-5분)
- 🔄 **새로고침**: 페이지를 새로고침하여 상태 확인

---

### **원인 2: 이미 배포됨** ✅

**증상**:
- "Published" 또는 "Deployed" 상태 표시
- URL이 생성되어 있음
- "Publish" 버튼 대신 "Update" 또는 "Republish" 버튼 표시

**해결**:
- ✅ **정상**: 이미 배포가 완료된 상태
- 🔄 **업데이트**: "Update" 버튼을 클릭하여 재배포

---

### **원인 3: 빌드 실패** ❌

**증상**:
- "Failed" 또는 "Error" 상태 표시
- 빨간색 오류 메시지
- "Retry" 버튼 표시

**해결**:
1. 오류 메시지 확인
2. 문제 수정
3. "Retry" 버튼 클릭

---

### **원인 4: 시스템 오류** 🐛

**증상**:
- 버튼이 계속 비활성화 상태
- 로딩 아이콘이 멈춤
- 아무 메시지도 표시되지 않음

**해결**:
1. 페이지 새로고침 (F5)
2. 브라우저 캐시 클리어
3. 다른 브라우저에서 시도
4. GenSpark 지원팀 문의

---

## ✅ 즉시 확인 방법

### **1단계: 현재 배포 상태 확인**

**Publish 탭에서 확인**:
```
✅ "Published" → 배포 완료
⏳ "Building" → 빌드 중 (기다리기)
❌ "Failed" → 배포 실패 (재시도)
```

### **2단계: 배포 URL 확인**

**Publish 탭에서 확인**:
```
✅ URL이 표시되어 있음 → 배포 완료
❌ URL이 없음 → 배포 미완료 또는 실패
```

예시 URL:
```
https://your-project.genspark.ai
```

### **3단계: 실제 사이트 접속 테스트**

배포된 URL로 접속하여:
- [ ] 메인페이지가 정상적으로 로드되는가?
- [ ] CSS가 적용되어 있는가?
- [ ] 이미지가 표시되는가?
- [ ] 네비게이션이 작동하는가?

---

## 🔧 문제 해결 단계

### **해결 방법 1: 페이지 새로고침** (가장 먼저 시도)

```
F5 키 또는 Ctrl+R
```

또는

```
브라우저 새로고침 버튼 클릭
```

---

### **해결 방법 2: 강제 새로고침**

```
Windows: Ctrl + Shift + R
Mac: Cmd + Shift + R
```

---

### **해결 방법 3: 브라우저 캐시 클리어**

**Chrome**:
1. Ctrl+Shift+Delete (또는 Cmd+Shift+Delete)
2. "캐시된 이미지 및 파일" 선택
3. "데이터 삭제" 클릭

**Firefox**:
1. Ctrl+Shift+Delete
2. "캐시" 선택
3. "지금 삭제" 클릭

---

### **해결 방법 4: 다른 브라우저에서 시도**

- Chrome
- Firefox
- Edge
- Safari (Mac)

---

### **해결 방법 5: 로그아웃 후 재로그인**

1. GenSpark에서 로그아웃
2. 브라우저 캐시 클리어
3. 다시 로그인
4. 프로젝트 열기
5. Publish 탭 확인

---

### **해결 방법 6: 프로젝트 재빌드**

**Publish 탭에서**:
1. "Unpublish" 또는 "Delete Deployment" 클릭
2. "Publish" 버튼 다시 클릭
3. 빌드 완료 대기

---

## 🎯 GenSpark Publish 정상 플로우

```
1️⃣ Publish 탭 열기

4️⃣ "Published" 상태 확인

### **업데이트 배포**:

```
1️⃣ 코드 수정
2️⃣ Publish 탭 열기

4️⃣ 재빌드 시작 (1-5분 대기)
5️⃣ "Published" 상태 확인
6️⃣ 배포 URL 새로고침하여 확인
```

## 🐛 빌드 실패 시 확인 사항

### **1. 파일 이름 및 경로**

**확인**:
- [ ] 특수 문자 사용 (한글, 공백, 특수기호)
- [ ] 대소문자 구분
- [ ] 경로 깊이 (너무 깊은 폴더 구조)

**권장**:
```
✅ index.html
✅ css/style.css
✅ pages/about/greeting.html

❌ css/스타일.css

---

### **2. 파일 크기**

**확인**:
- [ ] 이미지 파일이 너무 큰가? (5MB 이상)
- [ ] 전체 프로젝트 크기가 너무 큰가? (100MB 이상)

**권장**:
- 이미지 최적화 (WebP, 압축)
- 불필요한 파일 삭제

---

### **3. 문법 오류**

**확인**:
- [ ] HTML 문법 오류
- [ ] CSS 문법 오류
- [ ] JavaScript 문법 오류

**도구**:
- HTML Validator
- CSS Validator
- ESLint (JavaScript)

## 💡 팁: 배포 URL 직접 확인

버튼 상태와 관계없이:

1. Publish 탭에서 **배포 URL 복사**
2. 새 브라우저 탭에서 **해당 URL 접속**
3. **사이트가 정상 작동하는지 확인**

**배포 URL이 정상 작동한다면**:
→ 실제로는 배포 완료된 상태
→ 버튼 비활성화는 UI 버그일 가능성
